# Introduction to Leetcode

## Why do I need to know Leetcode?

Leetcode is essential to the process of landing a tech job. As Online Assessments (Hackerrank, CodeSignal, Codility, CoderPad, etc...) are used to filter candidates and Technical Interviews are used to test knowledge and further filter candidates to see how they perform under pressure.

It's also a great method to test your knowledge of DSA and problem-solving in a practical environment.

However, 99 out of a 100 you'll never use any of it. As in an enterprise environment the metrics they used are based on completely different optics.

So bear with the pain, to the knowledge that getting the job is harder than the job.

### Leetcode vs Other Platforms

Leetcode and other platforms like Neetcode are basically all the same just presented in a different way.

The reason I chose Leetcode over other platforms is just that Leetcode is the most well known and popular.

## How will this work?

Each Leetcode problem will have its own file including:
- Problem description
- Topics
- Hints
- Pseudocode solution.

Along with any notes/comments I might have had for the problem.

## My advice to solving problems

1. Read the problem description and it's testcases. 
2. When stuck click on topics, to give a small hint about what to do.
3. Click on the hints if still stuck, some will be useful others counterintuitive
4. Finally, compare your solution to mine

**Generally ignore the memory and runtime unless it specifies to use certain method, or you're doing an expanded problem**
